1.3 Slot Auto-Generation Logic
In Clinic model or a service function:

def generate_slots_for_date(clinic, date):
    """Auto-generate time slots for a given date based on clinic hours and services."""
    if not clinic.opening_time or not clinic.closing_time:
        return
    
    services = clinic.services.filter(is_active=True)
    if not services:
        return  # no services → no slots
    
    # For each service, create slots from opening_time to closing_time
    for service in services:
        current = datetime.combine(date, clinic.opening_time)
        end = datetime.combine(date, clinic.closing_time)
        
        while current + timedelta(minutes=service.duration_minutes) <= end:
            slot_end = current + timedelta(minutes=service.duration_minutes)
            
            # Don't duplicate if manually created slot exists
            existing = TimeSlot.objects.filter(
                clinic=clinic, date=date,
                start_time=current.time()
            ).first()
            if not existing:
                TimeSlot.objects.create(
                    clinic=clinic, date=date,
                    start_time=current.time(),
                    end_time=slot_end.time(),
                    is_available=True,
                    is_auto_generated=True,
                    service=service
                )
            
            current = slot_end
When to generate:

On clinic save (update closing_time or opening_time)
On service save (add/update duration)
Daily Celery task or on-demand via API
Doctor can manually trigger generation for a date range
When a slot is booked → is_available = False. When doctor toggles is_open = False → new bookings blocked, existing ones remain.

         │
         ▼
    Click a time slot → confirm modal
         │
         ▼
    Confirm booking → POST /api/appointments/
         │
         ▼
    Doctor gets notification
    Appointment status: PENDING
         │
         ▼
    Doctor confirms → status: CONFIRMED → patient notified
         │
         ▼
    Patient arrives → appointment completed → status: COMPLETED
